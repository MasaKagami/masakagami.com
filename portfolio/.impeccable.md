## Design Context

### Users
Portfolio visitors — recruiters, potential clients, collaborators, and fellow developers. They arrive to evaluate Masa Kagami's skills, aesthetic sensibility, and professionalism. They expect a fast, polished experience that reflects the craft of a frontend developer and product engineer.

### Brand Personality
**Minimal. Refined. Sophisticated.**

The portfolio should feel like a calm, deliberate statement — not a loud pitch. Every element earns its place. The design communicates creative confidence through restraint, not excess.

### Emotional Goals
- **Calm confidence** — the visitor feels they're looking at someone who knows what they're doing
- **Creative edge** — subtle animations and typographic choices hint at deeper craft
- **Quiet sophistication** — impressive without trying hard; the design speaks for itself

### Aesthetic Direction
- **Visual tone**: Editorial, monochromatic, grayscale with a warm off-white base
- **Background**: `#f3f3f3` — warm off-white, not sterile
- **Palette**: Strict grayscale — blacks, charcoals, medium grays, off-whites. No color accents.
- **Typography**: Lora (serif) for display/headings — elegant, editorial weight (400–500). Poppins (sans-serif) for body/UI text — clean, geometric, highly legible (400–500 weight).
- **Imagery**: Grayscale (`brightness-[0.65]`) with SVG `feTurbulence` film grain overlay masked to image alpha — evokes analog photography, adds texture and warmth to the monochrome palette. Images are `draggable={false}`.
- **Animations**: Sequenced, purposeful entrance animations. Image visible immediately, rolling text fades in while scrolling, logo mask-reveals from below, name text slides up, nav/roles stagger in with blur-to-sharp. Never bouncy, never playful. All respect `prefers-reduced-motion`.
- **Layout**: `max-w-7xl` content container. Generous whitespace, strong typographic hierarchy. The hero uses a slow-rolling serif background text with a large grayscale portrait.
- **Navigation**: No persistent navbar. A circle menu button (logo.svg, dark circle) appears after scrolling past 85% of the hero. Opens a full-screen dark overlay with Anime.js letter-reveal animations.
- **Text inversion effect**: Bottom role text uses a two-layer technique — dark text (`#1a1a1a`) below the image (covered by it), light text (`#f3f3f3`) inside the image container masked to the image's alpha channel. Only the exact pixels overlapping the photo invert. No blend modes, no brightness dependency.

### Reference
- **Primary reference**: [mahmoudashrf.vercel.app](https://mahmoudashrf.vercel.app/) — the editorial hero layout, grayscale palette, oversized typography, and clean section transitions
- **Key elements adopted**: Large serif rolling text behind a grayscale portrait, minimal navigation, generous spacing, role descriptors at bottom-left with color inversion on image overlap

### Anti-References
- No startup/SaaS landing page aesthetics (gradient blobs, floating UI mockups, badge walls)
- No playful or colorful portfolios (bright palettes, emoji, rounded bubbly shapes)
- No dark mode gamer/hacker aesthetics
- No template-looking designs with stock patterns

### Design Principles
1. **Restraint is confidence** — fewer elements, more impact. If it doesn't serve the message, remove it.
2. **Typography is the design** — type choices, scale, and spacing do the heavy lifting. Lean on Lora's editorial character for display, Poppins for UI.
3. **Grayscale is not boring** — texture (grain), contrast, whitespace, and the text inversion effect create visual richness without color.
4. **Motion earns attention** — every animation must feel intentional and subtle. Sequenced entrance with clear hierarchy: image → text → UI. No animation for animation's sake.
5. **Accessibility is non-negotiable** — WCAG AA compliance minimum. Sufficient contrast ratios, keyboard navigation, reduced motion support, semantic HTML, `aria-hidden` on decorative elements.

### Hero Section Architecture
```
Stacking (DOM order, no z-index except header):
  1. Rolling background text (absolute, slow horizontal scroll, opacity 0.07)
  2. Top header (relative, z-30 — logo + "Masa Kagami" left, nav/socials right)
  3. Border line (absolute bottom, invisible text spacer for height sync)
  4. Dark role text (absolute bottom — covered by image where they overlap)
  5. Image container (absolute bottom, centered):
     - Grayscale portrait (brightness 0.65)
     - Grain overlay (feTurbulence, masked to image alpha)
     - Light role text (#f3f3f3, masked to image alpha — visible on photo only)

Mobile: h-[85vh], no top-right nav (hidden sm:flex), no email
Desktop: h-screen, full nav + socials in top-right
```

### Animation Sequence
```
0.0s  — Image already visible (no animation)
0.3s  — Rolling text fades in (1.5s duration) while already scrolling
1.2s  — Logo fades in (0.6s)
1.35s — "Masa Kagami" slides up from behind overflow mask (0.7s)
1.8s  — Top-right nav rises with blur-to-sharp (0.7s)
1.8s  — Bottom role text rises with staggered children (100ms apart)
```
